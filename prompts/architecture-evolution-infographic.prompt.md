# Architecture Evolution Infographic — Meta-Prompt

## Visual Specification

A high-resolution 3D isometric diorama visualizing a vertical, three-tiered "Enterprise Intelligence" architectural stack.

**Environment:** 8K resolution, expansive bright light-grey background (#F5F5F7), soft studio lighting with subtle ambient occlusion, clean and modern enterprise technology aesthetic. No clutter — generous negative space around the central structure.

**Style:** Flat UI 2.0 aesthetics translated into polished 3D space. Utilize a strict color palette anchored by "State Blue" (#2563EB) and "Cortex Green" (#10B981), complemented by pristine white (#FFFFFF) and subtle metallic accents (brushed aluminum, matte silver connectors). Highly abstract, professional, and visually rich with clear spatial separation between tiers.

**Composition:** A vertical, interconnected multi-layer structural stack dominating the center, demonstrating a clear progression from bottom (foundation) to top (intelligence). Each tier floats with visible separation (glass-like connectors or luminous data conduits between layers). Camera angle: 30° isometric, slightly elevated to reveal all three platforms simultaneously.

## Layer Architecture (Bottom → Top)

### LAYER 1 — BOTTOM: Harness Layer (Reliability + Control)

**Color:** Deep State Blue (#2563EB) with white circuit-trace accents
**Visual metaphor:** A solid, fortified platform — think precision-machined base plate with embedded control circuits

**Elements on this tier:**
- **Central control loop** — A glowing circular track (feedback loop) with small data packets orbiting it, representing the harness's eval → optimize → deploy cycle
- **Permission gates** — Miniature 3D checkpoint barriers (like security turnstiles) at the platform edges, representing safety/permission layers
- **Tool slots** — Precisely machined rectangular bays (like USB-C ports rendered large) labeled abstractly, representing predefined tool interfaces
- **Context window** — A transparent bounded container (glass cube) with flowing text particles inside, representing managed context
- **Recovery nodes** — Small glowing blue spheres at corners with curved arrows, representing error recovery and retry logic

**Label plate:** "HARNESS — Reliability & Control" in clean sans-serif, anchored to the front edge
**Subtitle:** "Makes LLM output production-worthy"

---

### LAYER 2 — MIDDLE: Agent Runtime Layer (Learning + Autonomy)

**Color:** Cortex Green (#10B981) gradient transitioning to teal, with gold synaptic accents
**Visual metaphor:** An organic-mechanical hybrid — living circuitry, neural pathways merged with structured compute

**Elements on this tier:**
- **Skill library** — A rack of glowing green "skill crystals" (hexagonal prisms) arranged in a honeycomb shelf, representing reusable learned skills
- **Reflection loop** — A Möbius-strip-shaped track with a miniature agent figure traversing it, representing continuous self-evaluation
- **Memory banks** — Layered translucent storage slabs (like stacked glass drives) with data flowing between them: episodic → semantic → procedural
- **Learning signal** — Upward-spiraling particle stream (golden) emerging from completed tasks, feeding back into the skill library
- **Autonomy dial** — A physical dial/gauge rendered in 3D showing L0→L4 autonomy levels, needle pointing to L2-L3 range
- **Workflow forge** — A small anvil-like workstation where raw task inputs transform into optimized workflow patterns

**Label plate:** "AGENT RUNTIME — Learning & Autonomy" in clean sans-serif
**Subtitle:** "Makes LLM self-improving"

---

### LAYER 3 — TOP: Gateway Layer (Communication + Access)

**Color:** White and silver with electric blue (#60A5FA) accent lines — lightest, most airy tier
**Visual metaphor:** A communication hub/antenna array — open, broadcasting, connecting to the world

**Elements on this tier:**
- **Channel array** — Multiple upward-facing antenna dishes/broadcast nodes, each representing a messaging channel: WhatsApp (green dot), Teams (purple dot), Slack (magenta dot), SMS (yellow dot), Email (blue dot)
- **Message router** — A central switching node (like a miniature air-traffic-control tower) directing message particles to appropriate channels
- **Plugin ecosystem** — Orbiting satellite modules (small cubes) circling the gateway, representing extensible plugins and integrations
- **User presence indicators** — Small abstract human silhouettes at the edges, connected by luminous threads to the gateway
- **Multi-agent routing** — Forking pathways showing a single user request splitting into parallel agent dispatches

**Label plate:** "GATEWAY — Communication & Access" in clean sans-serif
**Subtitle:** "Makes LLM accessible everywhere"

## Inter-Layer Connections

**Between Harness ↔ Agent Runtime:**
- Thick luminous conduits (blue-to-green gradient) carrying "reliability signals" upward and "learning requests" downward
- Visual: structured data packets (cubes) flowing up; amorphous learning blobs flowing down and getting structured

**Between Agent Runtime ↔ Gateway:**
- Branching fiber-optic-style channels (green-to-white gradient) carrying "intelligent responses" upward and "user intents" downward
- Visual: refined diamond-shaped outputs flowing up; raw message bubbles flowing down

**Vertical spine:**
- A central glowing pillar running through all three layers — the "unified stack" backbone
- Label along spine: "Messaging → Gateway → Learning Agent → Tools → Memory → Reflection"

## Technical Rendering Notes

- **Depth of field:** Slight tilt-shift blur at extreme foreground/background to emphasize the central stack
- **Shadows:** Soft, diffused — no harsh directional shadows. Each tier casts a subtle shadow on the one below
- **Materials:** Matte surfaces with selective glossy accents (connectors, data conduits). No full chrome/mirror surfaces
- **Typography:** Inter or SF Pro Display for labels; monospace (JetBrains Mono) for technical annotations
- **Scale:** The middle Agent Runtime layer should be ~40% of total height (it's the most complex), with Harness at ~30% and Gateway at ~30%
- **Particle effects:** Subtle floating data motes (tiny luminous dots) in the air between layers, suggesting constant information flow
- **No text inside the 3D render itself** — all labels should appear as clean 2D overlays positioned precisely at layer edges
